What is HTTP you ask?

HTTP - HyperText Transfer Protocol

Http stands for hypertext transfer protocol.

Timothy John Berners-Lee

The founder of this protocol is Timothy John Berners-Lee. Timothy John Berners-Lee is a famous British scientist, inventor of the World Wide Web, and technologies such as URI, URL, HTTP, HTML. I think this man is really cool. The HTTP protocol is the basis of the World Wide Web we use it when viewing pages in a browser.

Versions Protocol HTTP:

  • HTTP 0.9 in 1992
  • HTTP 1.0 in 1996
  • HTTP 1.1 in 1999
  • HTTP 2.0 in 2015
There are several versions of this protocol. The first experimental version was called --HTTP zero point nine (0.9) was developed in nineteen ninety-two years (1992). The very first official version --HTTP one (1) was adopted as standard in nineteen ninety-six years (1996) And very soon the extended protocol version was adopted --HTTP one point one (1.1) in nineteen ninety-nine years (1999) this version is still used In two thousand and fifteen (2015), a new version of the protocol appeared. -- HTTP two (2) now this version is only being commissioned and does not support all browsers. All data on the Internet is transmitted via this protocol. And is a way to communicate between the client and the server.
How does this protocol work?
See it works quite simple. The mode of operation of this protocol is request-response.
If we want to make a request from the client to the server, then we need to say in some way what we need. It is also necessary to transfer additional information to the server. For example, we want to go to YouTube.com. In this case, we need to pass a method for how I make this request, this is a Get method, then the version of the protocol that I use this request is transmitted. Then I have to say where I am addressing on this server in this case YouTube.com.
And when the server understands this message,
it will be able to find the requested data and
send this data to the client in the form of a code and send a response similar to the one we sent when accessing the server.
Let's consider what methods can be used when a client accesses the server.

Methods HTTP

  • GET - the transfer request of the web page
  • POST - data transfer to web server
  • HEAD - requests only the page title
  • PUT - the room page to your web server
  • DELETE - deleting a page from a web server
  • TRACE - helps to track page changes
  • OPTIONS - helps to know what methods are supported
  • CONNECT - allows you to connect to the web server via Proxy
Most popular methods are: Method GET – is request to send a web page, and the POST method - transfer data to a web server for processing (for example, when we write comments to videos on YouTube) Further rarely used methods: method HEAD - requests only the page header (this is like the GET method only without the message body), method PUT - placing the page on a web server, DELETE - delete a page from the web server, method TRACE - helps to track changes to the page, namely what changes it makes, OPTIONS - helps to find out which methods are supported for a specific resource on a web server, CONNECT - allows you to connect to the web server through a proxy.
In response, the server sends information where the protocol field is indicated in the first field, which is used, then a three-digit number indicating the request processing status. There are the following statuses:

Statuses HTTP

  • 1XX - informational messages
  • 2XX - successful implementation (for example 200 OK)
  • 3XX - redirect
  • 4XX - error on the client side(for example 404 - page not found)
  • 5XX - server error
1xx - statuses starting with one are used to send informational messages 2xx - statuses that begin with the number two, indicate that the request was completed successfully 3xx - statuses that start with the number three speak of page redirection 4xx - statuses that start with the number four indicate that the error was made on the client side 5xx - statuses that start at number five indicate an error on the server side Several headers follow, such as the type of web page being transmitted, the length of the page, and the code of the web page. After the transfer of the web page, the connection is broken.
